# 编辑器侧栏悬浮反馈克制化设计

## 目标

将编辑器左侧文件树侧栏在鼠标悬浮时的视觉反馈，调整为更克制、更符合生产力工具常见实践的材质反馈。

## 当前状态

- 侧栏容器使用半透明毛玻璃底板
- 容器悬浮时会出现一条位于右侧内边缘的竖向渐变光带
- 光带通过 `after` 伪元素淡入，视觉上属于装饰性 hover 光效

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 整体方向 | 弱化为材质反馈 | 与现有低饱和、轻边界、轻交互的界面语言保持一致 |
| 光带处理 | 去掉显性竖向光带 | 避免容器 hover 比可点击元素更抢视线 |
| hover 反馈 | 仅保留轻微背景/边界变化 | 符合常见生产力工具的面板反馈模式 |
| 交互范围 | 只改侧栏容器 | 不改变按钮、文件树项等现有局部交互 |

## 方案对比

### 方案 A：保留光带但大幅减弱

优点：改动最小，保留少量精致感。  
缺点：本质仍是装饰性 hover 光效，和项目整体风格不完全一致。

### 方案 B：移除光带，改为极轻表面反馈

优点：最符合常见实践，整体更稳、更耐看。  
缺点：视觉存在感明显降低。

### 方案 C：完全取消容器 hover

优点：最克制。  
缺点：面板会偏静态，少一点层次。

## 最终选择

采用方案 B：移除显性竖向光带，只保留极轻的容器 hover 反馈。

## 实现方式

1. 抽出侧栏容器基础样式，集中表达当前设计意图
2. 删除 `after` 伪元素和对应 hover 淡入效果
3. 将 hover 调整为非常轻的背景与边框/内阴影变化
4. 保持动画存在，但降低存在感，避免“亮一下”的感受

## 实施补充

- 视觉复查后确认，用户主观感受到的“扫光”有一部分实际上来自侧栏右侧的拖拽分隔条 `ResizeHandle`
- 因此同样采用一致策略：移除径向 glow 和跟随指针的热点，只保留极轻的竖线提示
- 这样侧栏容器与分隔条都遵循同一套克制化材质反馈语言

## 验证标准

- 正常状态和悬浮状态仍有微弱差异
- 不刻意观察时，hover 光效不应成为注意力焦点
- 亮色和暗色主题下都不显得突兀
- 顶部按钮与文件树项仍是主要交互反馈来源
